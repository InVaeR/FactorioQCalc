# SPEC.MD

## Калькулятор качества Factorio на PySide6

> ✅ — реализовано 🚧 — в разработке 📋 — запланировано

### 1. Назначение

Десктопное приложение на **PySide6**, которое рассчитывает ожидаемое
распределение выхода продукции по уровням качества в Factorio. Пользователь
задаёт параметры установки (количество и характеристики модулей качества, объём
производства, максимально доступный уровень качества), а калькулятор показывает,
сколько продукции каждого уровня качества будет произведено. ✅

**Расширение:** поддержка цикла рециклинга — учёт переработки продукции низкого
качества обратно в производство. ✅

---

### 2. Игровая механика (модель расчёта)

В Factorio существует 5 уровней качества:

| Уровень | Название                | Множитель |
| ------- | ----------------------- | --------- |
| 1       | Обычное (Normal)        | 1×        |
| 2       | Необычное (Uncommon)    | —         |
| 3       | Редкое (Rare)           | —         |
| 4       | Эпическое (Epic)        | —         |
| 5       | Легендарное (Legendary) | —         |

> **Разблокировка качества.** В игре уровни качества выше Uncommon разблокируются через технологии. Если уровень качества ещё не открыт, продукт не может на него повыситься — каскад улучшения «упирается» в максимально открытый уровень, а излишек остаётся на нём (аналогично потолку легендарного).

**Шанс улучшения качества от модулей.** Базовый бонус качества зависит от уровня (tier) модуля и его собственного качества:

| Модуль качества  | Бонус (по качеству модуля: Normal / Uncommon / Rare / Epic / Legendary) |
| ---------------- | ----------------------------------------------------------------------- |
| Quality Module 1 | 1.0% / 1.3% / 1.6% / 1.9% / 2.5%                                        |
| Quality Module 2 | 2.0% / 2.6% / 3.2% / 3.8% / 5.0%                                        |
| Quality Module 3 | 2.5% / 3.2% / 3.9% / 4.6% / 6.2%                                        |

**Суммарный шанс качества** = сумма бонусов всех установленных модулей качества.

**Правило каскадного улучшения:**

- С вероятностью `Q` (суммарный шанс) продукт повышается на уровень качества.
- При повышении с вероятностью 10% продукт повышается ещё на один уровень (и так далее каскадом). При срабатывании улучшения 90% попадает на следующий уровень, 9% — через один, 0.9% — ещё выше и т.д.
- **Потолок = максимально открытый уровень качества** (`max_unlocked_level`). Выше него продукт не поднимается, весь излишек каскада накапливается на этом уровне.

**Формула распределения для одного прохода:**

- Доля, оставшаяся на исходном уровне `i`: `1 − Q`
- Доля, поднявшаяся на `i+1`: `Q × 0.9`
- На `i+2`: `Q × 0.09`
- На `i+k`: `Q × 0.9 × 0.1^(k-1)`
- Накопление остатка на уровне `max_unlocked_level` (всё, что попало бы выше потолка, остаётся на нём).

> **Уточнение:** калькулятор рассчитывает результат как для **одного прохода** производства (без рециклинга) ✅, так и для **установившегося режима с рециклингом** ✅ (см. п. 7).


- Шанс не должен превышать **100%** (валидация). ✅
- Каскад улучшения ограничен сверху `max_unlocked_level`. ✅
- Если `start_level == max_unlocked_level`, весь выход остаётся на этом уровне (улучшаться некуда). ✅
- Если `start_level > max_unlocked_level` — ошибка валидации. ✅
- Все доли нормализуются так, чтобы сумма = 100%. ✅

#### Расчёт с рециклингом ✅

**Модель:** рециклинг в Factorio возвращает 25% материала (потеря 75%). Если в
рециклере установлены модули качества, они применяются к возвращаемому
материалу.

**Математика установившегося режима:**

Пусть:

- `Q_prod` — суммарный шанс качества в производстве
- `Q_rec` — суммарный шанс качества в рециклере
- `R` — порог рециклинга (уровни ≤ R идут в рециклер)
- `M` — `max_unlocked_level`
- `dist(start, Q, rate, M)` — функция распределения для одного прохода
- `base_input` — сырьё, подаваемое на вход системы

**Итеративный расчёт по очереди (очередь по уровням качества):**

```
queue = {start_level: base_input}
final = {}
for each iteration:
    craft всё из queue через distribute() с Q_prod
    for each level lvl:
        if lvl <= R:
            recycle через distribute(lvl, Q_rec, amount * 0.25)
            результат → queue следующей итерации
        else:
            добавить к final
    если queue пуст → сходимость
```

**Конечный результат:** `final` — суммарный выход каждого уровня качества.

**Особенности:**

- Рециклинг может повышать качество материалов за счёт качества в рециклере
- Потери 75% на каждом цикле рециклинга
- Система всегда сходится, т.к. каждый цикл теряет 75% массы
- Без модулей в рециклере (`Q_rec = 0`) возвращается материал того же качества
- Если `R == 1` (порог = Normal) — всё, кроме Normal, уходит на выход

### 9. Критерии приёмки

*Один проход:* ✅

1. При 0 модулей весь выход остаётся на начальном уровне. ✅
2. Сумма долей всегда равна 100% (±0.01%). ✅
3. Каскад улучшения **не превышает** `max_unlocked_level`; весь излишек накапливается на нём. ✅
4. При `start_level == max_unlocked_level` весь выход на этом уровне. ✅
5. При `start_level > max_unlocked_level` приложение показывает ошибку валидации. ✅
6. Изменение «Макс. открытое качество» корректно пересчитывает таблицу и ограничивает число отображаемых строк. ✅
7. Юнит-тесты на `core/calculator.py` (включая случаи с разными `max_unlocked_level`, `total_chance_from_modules`, баланс массы, валидацию рециклинга) проходят. ✅ (45 тестов)
8. UI корректно отображает результат после нажатия «Рассчитать». ✅

*Рециклинг:* ✅

9. При выключенном рециклинге поведение идентично «одному проходу». ✅
10. При `Q_rec = 0` рециклер возвращает 25% материала без изменения качества. ✅
11. Сумма долей на выходе = 100% (±0.01%). ✅
12. Система сходится за конечное число итераций (независимо от начальных параметров). ✅
13. При `Порог рециклинга > Макс. открытое качество` — ошибка валидации. ✅
14. При `Порог рециклинга == Normal` и выключенных модулях рециклера — Normal в рециклер, остальное на выход. ✅

## Style guide

### Основное описание

Создай интерфейс в стиле игры Factorio — индустриальный, технический UI с акцентом на функциональность и "механическое" ощущение элементов управления. Дизайн должен передавать атмосферу фабрики, автоматизации и инженерных систем.

### Цветовая палитра

**Основные цвета:**

- Фон мира/приложения: `#1a1a1a` (почти черный)
- Фон панелей: `#2b2b2b` (темно-серый)
- Заголовки/акценты: `#3a3a3a` (серый)

**Акцентные цвета:**

- Оранжевый (операторы, важные действия): `#d97b14` (основной), `#f09030` (светлый), `#8a4a00` (темный)
- Зеленый (подтверждение, успех): `#4a8c1c` (основной), `#6bb32e` (светлый), `#2a520d` (темный)
- Красный (удаление, ошибка, опасность): `#8c1c1c` (основной), `#b32e2e` (светлый), `#520d0d` (темный)
- Желтый (заголовки, предупреждения): `#ffcc00`

**Текст:**

- Основной текст: `#e0e0e0` (светло-серый)
- Второстепенный текст: `#a0a0a0` (серый)

### Типографика

**Шрифты:**

- Системные шрифты без засечек: Segoe UI, Roboto, Helvetica Neue, Arial
- Жирное начертание для кнопок и заголовков (font-weight: bold)
- Увеличенный межбуквенный интервал для заголовков (letter-spacing: 1-2px)

**Размеры:**

- Заголовки: 12-14px, uppercase, letter-spacing: 1px
- Кнопки: 16-20px, bold
- Дисплеи/ввод: 24-32px, bold
- Основной текст: 14-16px

### Эффекты и границы (Bevel Effect)

**Выпуклые элементы (кнопки, панели):**

```
border-top: 2px solid [светлый цвет]
border-left: 2px solid [светлый цвет]
border-bottom: 2px solid [темный цвет]
border-right: 2px solid [темный цвет]
```

**Вдавленные элементы (дисплеи, поля ввода):**

```
border-top: 2px solid [темный цвет]
border-left: 2px solid [темный цвет]
border-bottom: 2px solid [светлый цвет]
border-right: 2px solid [светлый цвет]
```

**Состояния кнопок:**

- **Normal:** Выпуклый вид со светлыми границами сверху/слева
- **Hover:** Осветление основного цвета на 10-15%, более яркие светлые границы
- **Pressed:** Инверсия границ (темные сверху/слева, светлые снизу/справа), смещение текста на 1-2px вниз-вправо для эффекта "утопленности"

### Структура панелей

**Контейнеры:**

- Толстая граница (2-3px) с эффектом bevel
- Скругление углов минимальное (2-3px) или отсутствует
- Внутренние отступы: 8-12px
- Отступы между элементами: 4-8px

**Заголовки панелей:**

- Отдельная секция сверху с фоном `#3a3a3a`
- Текст по центру, uppercase, желтый цвет `#ffcc00`
- Нижняя граница для отделения от содержимого

### Поведение и анимации

**Переходы:**

- Мгновенные изменения состояний (без плавных анимаций)
- Визуальная обратная связь через изменение цветов и границ

**Интерактивность:**

- Все кликабельные элементы должны иметь четкие состояния hover/pressed
- Курсор при наведении на кнопки: pointer
- Фокус на элементах ввода: изменение цвета границы на оранжевый

### Дополнительные элементы

**Иконки:**

- Простые, геометричные, монохромные
- Толщина линий: 2-3px
- Цвет: соответствует тексту или акцентному цвету

**Скроллбары:**

- Темный фон трека: `#1a1a1a`
- Светлый ползунок: `#4a4a4a` с bevel-эффектом
- Минимальная ширина: 12-16px

**Таблицы/списки:**

- Чередование фона строк: `#2b2b2b` / `#333333`
- Разделители: 1px линия `#0a0a0a`
- Выделение строки: оранжевая левая граница 3px

### Ключевые принципы

1. **Функциональность превыше декора** — каждый элемент имеет четкое назначение
2. **Механическое ощущение** — элементы выглядят как физические кнопки и переключатели
3. **Высокий контраст** — четкое различие между активными и неактивными элементами
4. **Индустриальная эстетика** — ассоциации с заводским оборудованием и панелями управления
5. **Читаемость** — текст и иконки должны быть хорошо видны на темном фоне

- Калькуляторы и панели ввода данных
