## 2. Архитектура

Приложение разделено на независимые слои:

```
core/calculator.py  ←→  core/data.py         # Чистая логика, без Qt
        ↕
ui/main_window.py   ←→  assets/images/*.png  # UI + визуальные ресурсы
        ↕
main.py             ←→  QApplication         # Точка входа
```

- **`core/`** не импортирует PySide6 и может быть покрыт тестами
- **`ui/`** использует Qt для отображения и вызывает функции `core/`
- **`assets/`** содержит иконки и графику для Factorio-стиля

## 3. Ядро расчёта

### `core/data.py`
- `QUALITY_NAMES` — английские названия уровней (1–5)
- `QUALITY_NAMES_RU` — русские названия
- `TIER_NAMES` — названия модулей
- `BONUS_TABLE` — таблица бонусов `[tier][quality]`
- `MAX_QUALITY_LEVEL = 5`

### `core/calculator.py`

**`quality_bonus(tier, quality) → float`**
- Возвращает бонус одного модуля (0–1)
- `tier`: 1, 2, 3
- `quality`: 1 (Normal) … 5 (Legendary)
- Исключение при невалидных параметрах

**`total_chance(count, tier, quality) → float`**
- Суммарный шанс качества = `count × bonus`, ограничен 1.0

**`distribute(start_level, chance, output_per_min, max_unlocked_level) → dict`**
- Возвращает `{level: (share, items_per_min)}`
- Каскадное улучшение: 90% на следующий уровень, 10% на каскад далее
- Потолок: `max_unlocked_level` (весь излишек накапливается на нём)
- Если `start_level == max_unlocked_level` → 100% остаётся на месте

#### Формула каскада

Для уровня `start_level = s` и `max_unlocked_level = M`:

| Уровень | Доля |
|---------|------|
| `s` | `1 − Q` |
| `s + k` (для `k < M−s`) | `Q × 0.9 × 0.1^{k−1}` |
| `M` | `Q × 0.1^{M−s−1}` |

Где `Q` — суммарный шанс качества.

## 5. Рециклинг

### UI
- `QCheckBox` «Рециклинг» — включает/отключает режим
- Поля рециклинга скрыты, пока не включен чекбокс:
  - `Порог рециклинга` — всё, что ниже этого уровня, идёт в рециклер
  - `Кол-во модулей рец.` — модули качества в рециклере
  - `Уровень модуля рец.` — тир модулей рециклера
  - `Качество модуля рец.` — качество модулей рециклера
- Дополнительная строка в результатах: коэффициент рециклинга + шанс рециклера

### Алгоритм (`calculate_with_recycling`)

Итеративный расчёт установившегося режима:

```
queue = {start_level: base_input}
final = {}
for each iteration:
    craft всё из queue через distribute() с Q_prod
    for each level lvl:
        if lvl < recycle_threshold:
            recycle через distribute(lvl, Q_rec, amount * 0.25)
            результат → queue следующей итерации
        else:
            добавить к final
    если queue пуст — сходимость
```

**Ключевые свойства:**
- Каждый цикл рециклинга теряет 75% массы
- Система гарантированно сходится (убывающая геометрическая прогрессия)
- Качество модулей рециклера повышает качество возвращаемых материалов
- При `Q_rec = 0` возвращается 25% материала без изменения качества
- При `threshold = 1` рециклинг де-факто отключён (все уровни ≥ 1 выходят)

## 6. Тесты

28 тестов в `tests/test_calculator.py`:
- `TestQualityBonus` (6 тестов) — корректность бонусов, невалидные параметры
- `TestTotalChance` (5 тестов) — суммирование, кап на 100%, отрицательное кол-во
- `TestDistribute` (9 тестов) — нулевой шанс, potолок, сумма долей = 100%, распределение для разных `max_unlocked_level`, проверка `start_level > max_unlocked_level`
- `TestRecycling` (8 тестов) — `threshold=1` эквивалентен single pass, нулевой `Q_rec`, конвергенция, улучшение от качества рециклера, валидация

Запуск: `pytest tests/test_calculator.py -v`
